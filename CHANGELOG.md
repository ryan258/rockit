# **CHANGELOG**

## **Phase 0: Audio Normalization (Suno Pre-Processing)**

_Critical: Suno tracks have organic, fluctuating tempos. VR rhythm games require a mathematically perfect, static BPM grid. Skipping this step results in extreme note drift._

- **Action:** Download the .mp3 or .wav from Suno.
- **Automated (`warp.sh`):** Uses an AI stems tool (Demucs), beat tracking (Librosa), and time-stretching (Rubberband) to mathematically lock the track to a **fixed, static BPM** and export it as a `.wav` file.

## **Phase 1: Schema Mapping (The Translation Matrix)**

You will be parsing JSON arrays. Here is the exact data mapping your Python script needs to execute.

### **Source: Beat Saber Note (v2/v3 from Beat Sage)**

Beat Saber uses 4 columns (0,1,2,3).

- \_time or b: The beat number (Float). \-\> **KEEP**
- \_lineIndex or x: The column (0=Left, 3=Right). \-\> **KEEP**
- \_lineLayer or y: The height row. \-\> **DROP**
- \_type or c: Note color. \-\> **DROP**
- \_cutDirection or d: Saber swing direction. \-\> **DROP**

### **Target: Ragnarock Note**

Ragnarock uses 4 drums (0,1,2,3).

- \_time: The beat number (Float).
- \_lineIndex: The drum (Integer 0-3).

## **Phase 2: Core Python Pipeline (Antigravity IDE)**

Your local script (rr_converter.py) is structured into three core functions:

### **1\. File Extraction (extract_bs_data)**

- **Action:** Accept a .zip file generated by Beat Sage.
- **Logic:** Unzip to a temporary directory. Read Info.dat to extract the \_beatsPerMinute and audio file name. Read Expert.dat (or whichever difficulty) to extract the \_notes array.

### **2\. The Conversion Engine (convert_notes)**

- **Action:** Iterate through the Beat Saber \_notes array, standardizing v2 and v3 Beat Saber schema into a generic time/lane format.

### **3\. The MS Playability Filter (clean_chart)**

_Critical for energy conservation and avoiding AI "noise" or impossible drum rolls._

- **Deduplication:** Beat Saber often stacks notes on different layers in the same column. If two notes exist at the exact same time and lane, delete one.
- **Hammer Limit:** A player only has 2 hammers. If the AI generated 3 or 4 notes on the exact same beat, **drop the extras** (keep only the outer two).
- **Speed Cap:** Calculate the time delta between notes. If notes are clustered closer than 0.125 beats (1/32nd notes), prune them to prevent physical exhaustion from AI-generated spam.

## **Phase 3: Metadata & Packaging**

Ragnarock requires a specific Info.dat structure to load the song properly. The script generates a new Info.dat alongside the mapped Difficulty.json.

### **Required Ragnarock Info.dat Injection:**

- \_songName: Extracted from Beat Saber info.
- \_beatsPerMinute: Extracted from Beat Saber info.
- \_environmentName: "Midgard"
- \_difficultyBeatmapSets: Array pointing to the generated JSON (Level1.json).

### **Output Generation (package_rr_song)**

- **Action:** Copy the original .ogg audio file and .jpg cover art from the unzipped Beat Saber folder.
- **Action:** Save the new Info.dat and Level1.json in the same directory.

## **Phase 4: Execution Flow**

1. Run `./warp.sh` to lock the BPM of the raw audio.
2. Upload BPM-locked `.wav` to Beat Sage. Download `.zip`.
3. Run `./rockit.sh "beat_sage_output.zip"`
4. Script outputs `[SongName]_Ragnarock` folder.
5. Move folder to Quest: Android \> data \> com.wanadev.ragnarockquest \> files \> UE4Game \> Ragnarock \> Ragnarock \> Saved \> CustomSongs
